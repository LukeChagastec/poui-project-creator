# feat(core): atualiza o projeto para Angular 21

## Resumo
Atualiza a estrutura do projeto gerado para utilizar o Angular 21, garantindo compatibilidade com as versões mais recentes do ecossistema Angular e PO UI. Esta atualização moderniza a base do projeto, introduz o novo builder de aplicação do Angular e ajusta as dependências para resolver conflitos de instalação.

## ⚙️ Detalhes Técnicos Aprofundados
### 📂 TypeScript
- **`src/extension.ts`**:
- **Problema/Motivação**: A versão anterior do gerador de projetos estava fixada no Angular 19, o que resultava em dependências desatualizadas e conflitos durante a instalação em ambientes de desenvolvimento modernos.
- **Implementação**:
    - O comando `ng new` foi atualizado para usar `npx -p @angular/cli@21`.
    - As dependências do PO UI (`@po-ui/ng-components`, `@po-ui/ng-templates`) foram atualizadas para as versões mais recentes compatíveis com o Angular 21.
    - Foi adicionada a função `configurePackageJson` para ajustar programaticamente as versões das dependências no `package.json`, incluindo a adição de uma seção `overrides` para forçar a resolução de pacotes aninhados do Angular.
    - O comando `npm install` agora utiliza a flag `--force` para superar bloqueios de instalação.
- **Impacto**: Projetos gerados agora são baseados no Angular 21, com um processo de instalação de dependências mais robusto e menos propenso a erros de peer dependency.

- **`src/templates/app-component-ts.ts.template`**:
- **Problema/Motivação**: O template do `AppComponent` referia-se a arquivos `.html` e `.css` que não são o padrão gerado pelo Angular CLI em versões mais recentes.
- **Implementação**: O `templateUrl` foi alterado de `./app.component.html` para `./app.html` e `styleUrls` de `['./app.component.css']` para `['./app.css']`.
- **Impacto**: Garante que o componente principal da aplicação funcione corretamente com a estrutura de arquivos gerada pelo Angular CLI 21.

- **`src/templates/env-prod.ts.template`**:
- **Problema/Motivação**: O ambiente de produção não estava herdando as configurações do ambiente local, o que poderia levar a inconsistências.
- **Implementação**: O `localEnvironment` agora é importado e suas propriedades são espalhadas (`...localEnvironment`) no objeto `environment` de produção.
- **Impacto**: Unifica as configurações de ambiente, permitindo que as configurações locais sirvam de base para o ambiente de produção.

### 📂 Configuração
- **`.gitignore`**:
- **Problema/Motivação**: O arquivo `ver.md`, que contém as regras de versionamento, não deve ser versionado no repositório.
- **Implementação**: Adicionado `ver.md` ao `.gitignore`.
- **Impacto**: Evita que o arquivo de regras de versionamento seja incluído nos commits.

- **`angular.json` (modificado por `src/extension.ts`)**:
- **Problema/Motivação**: O builder do Angular foi alterado nas versões mais recentes, e a configuração do `angular.json` precisava ser atualizada.
- **Implementação**: O builder foi alterado para `@angular-devkit/build-angular:application` e a estrutura de `build.options` foi ajustada para o novo formato.
- **Impacto**: Alinha a configuração de build do projeto com as práticas mais recentes do Angular, permitindo o uso de recursos de build otimizados.
